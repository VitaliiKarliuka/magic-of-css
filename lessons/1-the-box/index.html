Lesson 1 The Box

Box model

In CSS, the box model describes the rectangular boxes that are generated for elements layed out in the page.

Some interesting facts:

  • border-radius rounds out the corners of this box
  • box-shadow adds a shadow to this box
  • oultine and box-shadow aren't part of the box, and therefore have no effect on the layout

box-sizing

The box-sizing property gives you a little control around how boxes are sized within this model. The two possible values for box-sizing are content-box and border-box.

content-box
The default. When computing the size of a box, padding and border are added.
border-box
When computing the size of a box, padding and border are folded in.

Both of these boxes have the following CSS:

.box {

border-box

In the border-box case, the width and height of the .box are 20px, exactly what we set. In the content-box case, the width and height are 44 = 20 + (2 * 10) + (2 * 2), since we need to include the padding and border on both sides.

tl;dr for this lesson

If you height and width to behave in the most intuitive way, listen to Paul Irish and put this at the top of yur CSS:

* {
*, *:before, *:after {
    -webkit-box-sizing: border-box; /* Needed for mobile webkit */
    -moz-box-sizing: border-box; /* Needed for Firefox */
    box-sizing: border-box
}
